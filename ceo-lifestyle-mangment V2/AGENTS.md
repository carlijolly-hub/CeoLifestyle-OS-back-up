# AGENTS.md - Persistent Developer and Documentation Guidelines

This file outlines persistent project rules, operational constraints, and workflow patterns for AI Coding Agents working on this repository. Do NOT delete or modify existing rules unless explicitly requested by the Master Administrator.

---

## Master Administrator Guide Update Control

The **Master Administrator Guide** inside the Branding Settings (`/src/components/BrandingSettings.tsx`) should **not automatically update** after every individual application change or update.

The guide must only be updated when specifically instructed by the Master Administrator. Before adding documentation updates, **always wait for explicit approval**.

### 1. Documentation Update Rules

Only create or modify Master Administrator Guide entries when:
* The Master Administrator explicitly requests a guide update.
* A major version release is being documented.
* A significant module or workflow change requires documentation.

Do **NOT** automatically create guide entries for:
* Minor feature adjustments.
* Bug fixes.
* UI changes.
* Layout improvements.
* Small functionality updates.
* Testing changes.

### 2. Version Documentation Process

When instructed to update the Master Administrator Guide, include:
* **Version number**
* **Date of update**
* **Feature/module affected**
* **Description of the change**
* **Purpose of the change**
* **How to access the feature**
* **How to use the feature**
* **Any workflow changes**

### 3. Final Objective

Keep the Master Administrator Guide as a curated, high-value operating manual rather than an automatic log of every development change. Documentation should remain clean, organized, and meaningful, with updates controlled by the Master Administrator.
